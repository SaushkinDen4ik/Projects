# Стэк-аллокатор и упрощённая версия std::list

В данной задаче реализован класс стэк-аллокатор, который бы позволял создавать стандартные контейнеры на стеке, без единого обращения к динамической памяти
Стэк-аллокатор обращается к памяти через пулл.
Преимущество использование собственного аллокатора заключается в его скорости по сравнению с std::allocator (замеры по времени производились)
Также реализован шаблонный класс List, реализованный как std::list через вершины, где BaseNode - тип для фейковой вершины, существующий тут потому что фейк
вершина не хранит значение. Хранит значение Node, структура, унаследованная от BaseNode
